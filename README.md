# Applied Cryptography: RAG Database Encryption Engine

> *Repositorio público duplicado para exhibición en portafolio, libre de restricciones académicas de derechos de autor o versiones.*

## Title & Description
Retrieval-Augmented Generation (RAG) systems heavily rely on vector databases to feed domain-specific context to Large Language Models (LLMs)[cite: 10]. However, when these databases ingest sensitive documents (e.g., medical records containing Personally Identifiable Information - PII), the physical storage layer becomes a massive security liability[cite: 10].

This project implements a robust cryptographic architecture for **Encryption at Rest**, ensuring that if physical drives or cloud buckets hosting the vector database are compromised, the raw data remains cryptographically secure and inaccessible without the proper authorization keys[cite: 10].

## Architecture & Technical Implementation

The repository contains a fully functional Proof of Concept (PoC) in Python demonstrating a secure, industry-standard cryptographic pipeline:

### 1. Envelope Encryption Strategy (Key Management)
To prevent the catastrophic failure of storing decryption keys alongside the ciphertext, this architecture implements an Envelope Encryption protocol:
* **Key Encryption Key (KEK):** A 256-bit master key managed by a simulated Key Management Service (KMS). It never touches the raw data[cite: 10].
* **Data Encryption Key (DEK):** A fast, symmetric 256-bit key that performs the heavy lifting of encrypting the actual dataset[cite: 10]. 
* **The Process:** The KMS generates the DEK, wraps (encrypts) it using the KEK, and sends both the wrapped DEK and the plaintext DEK to the storage module[cite: 10]. After encrypting the data, the plaintext DEK is purged from memory. On disk, only the `encrypted_database.bin` and the locked `encrypted_dek.bin` exist[cite: 10].

### 2. Mode of Operation: AES-GCM
The actual data encryption utilizes the Advanced Encryption Standard in Galois/Counter Mode (AES-GCM)[cite: 10]:
* **Confidentiality:** Employs a counter-driven stream cipher approach to obscure data patterns, unlike vulnerable modes such as ECB[cite: 10].
* **Integrity (AEAD):** AES-GCM appends an authentication tag (MAC) generated via universal hashing over a binary Galois field. During the decryption phase, the system verifies this tag to guarantee the ciphertext has not been tampered with or maliciously altered[cite: 10].

### 3. Academic Research Report
The repository includes a highly detailed technical report (`report.pdf` / `report.typ`) authored using Typst[cite: 10]. It breaks down the theoretical necessity of the architecture, the specific mechanics of the Envelope Strategy, and the cryptographic advantages of AEAD ciphers in RAG environments[cite: 10].

## Tech Stack
* **Language:** Python 3[cite: 10].
* **Cryptography:** `cryptography.hazmat.primitives.ciphers.aead.AESGCM`[cite: 10].
* **Data Simulation:** Secure generation of IVs (Initialization Vectors) and keys using `os.urandom`[cite: 10].
* **Typesetting:** Typst (Academic layout engine)[cite: 10].

## Key Challenges & Learnings
1. **AEAD Tag Management:** Correctly detaching and re-attaching the 16-byte authentication tag from the ciphertext blob generated by the `cryptography` library to ensure smooth binary file saving and successful decryption validation[cite: 10].
2. **Key Isolation:** Designing the software architecture (`MockKMS` vs. `VectorDBStorage` classes) to strictly enforce separation of concerns, ensuring the KEK never leaves the KMS layer[cite: 10].
3. **Binary File Handling:** Managing raw byte data (`b"..."`) efficiently for reading the CSV, executing the encryption, and writing the structured `.bin` files (IV + Tag + Ciphertext)[cite: 10].

## Installation & Usage

1. **Clone the repository:**
```bash
git clone [https://github.com/nico-maire/rag-database-encryption.git](https://github.com/nico-maire/rag-database-encryption.git)
cd rag-database-encryption

```

2. **Install dependencies:**
The only external requirement is the official Python cryptography toolkit.

```bash
pip install cryptography

```

3. **Run the Proof of Concept:**
The script will automatically detect `pii_dataset.csv`, spin up the mock KMS, execute the Envelope Encryption pipeline, save the `.bin` files, and then simulate a successful decryption and validation process.



```bash
python envelope_encryption_poc.py

```
